# Attribution

## AI Development Tools

This project used AI assistance extensively during development. The following describes what was generated, what was modified, and what required manual debugging or rework.

### Claude Code (Anthropic)

**Used for:** The web-facing serving layer ‚Äî `serving/triplet_inference/app.py`, `serving/dinov3_endpoint/model_server.py`, `serving/triplet_inference/static/index.html`, and the Dockerfile.

**What was generated:** The HTML/CSS/JS in `index.html` (drag-and-drop zone, pipeline timeline with confidence bars, color-coded decision card) was largely generated by Claude Code. The overall skeleton of `app.py` ‚Äî `read_triplet_zip`, the `/api/predict-triplet` endpoint structure, and the `stage()` helper ‚Äî was drafted with Claude Code assistance. `model_server.py`'s `MultiModelService` class structure and GCS artifact resolution were similarly scaffolded.

**What was modified and debugged manually:** The `call_gemini` function required manual rework to correctly pass `types.Part.from_bytes` image parts to the Gemini SDK. The `engineer_features` / `engineer_features_v2` functions were hand-written to exactly match the feature schema from the training notebooks. Confidence extraction logic (`confidence_for_occupancy`, `confidence_for_clean_dirty`) and the `normalize_perception` multi-format handler were written and debugged manually. The `sys.path.insert` import fix for sharing `model_server.py` across both serving apps was a manual fix.

**What was not AI-generated:** Both training notebooks (`dino_v3_rerun_(1) (1).ipynb`, `simple_yolo_classifier_occ_unocc.ipynb`) were written manually. All ML design decisions ‚Äî model architecture choices, Focal Loss with label smoothing, differential learning rates, warmup + cosine schedule, GroupShuffleSplit leakage prevention, feature engineering, ablation study design, ensemble construction ‚Äî were made and implemented by hand without AI code generation.

### Google Gemini (via AI Studio)

**Used for:** Multimodal escalation at inference time (not development). Gemini 2.5 Flash is called as an API within the pipeline when local model confidence falls below the 80% threshold. This is runtime tool use, not code generation.

### GitHub Copilot / general autocomplete

**Used for:** Minor boilerplate completions (imports, standard sklearn patterns) in notebook cells. No ML design decisions were AI-driven.

## Data

- Training images and `perception.json` files were collected from the Shire restaurant environment using an internal perception simulation pipeline (`table_perception_simulator.html`).
- No external datasets were used.

## Model Weights

- **DINOv3-ViT-Large backbone:** `facebook/dinov3-vitl16-pretrain-lvd1689m` loaded via HuggingFace `transformers`. Pre-trained weights are property of Meta AI, used under the Apache 2.0 license.
- **Fine-tuned classifier head:** trained from scratch on project data.
